%------------------------------------------------------------------------------
%
% PRIORITY: Clarity over brevity. These are learning notes, not cheat sheets.
%
% 1. BE DETAILED IN PROCESS:
%    - Show every step in derivations (don't skip algebra)
%    - State what you're doing before you do it ("By LOTP:", "Rearranging:")
%    - Use LHS/RHS labels when comparing expressions
%    - Explain WHY each step works, not just WHAT it is
%
% 2. BE CLEAR IN EXPLANATIONS:
%    - Use plain English, avoid jargon abbreviations (not "W.p." but "with probability")
%    - State goals explicitly ("Goal: Show that X does not imply Y")
%    - Add intuition boxes after formal proofs
%    - Use concrete examples with actual numbers
%
% 3. STRUCTURE FOR UNDERSTANDING:
%    - Label steps: "Step 1:", "Step 2:", etc.
%    - Use parallel structure (if example A has 4 steps, example B should too)
%    - Put key formulas in boxes, but show the work to get there
%    - End with "Intuition:" or boxed takeaway
%
% 4. WHAT TO AVOID:
%    - Cryptic abbreviations without definition
%    - Skipping "obvious" steps (nothing is obvious when learning)
%    - Dense walls of math without prose guidance
%    - Assuming the reader remembers earlier definitions
%    - Em dashes (---). Use colons, semicolons, or break into sentences instead
%
